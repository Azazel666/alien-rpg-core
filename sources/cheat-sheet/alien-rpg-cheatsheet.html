Copy and paste these HTML snippets into your chapter files. Replace the placeholder text with content from your PDF.

1. CHAPTER START - HERO BOX

Use this at the beginning of each chapter. Creates a hero box with horizontal dividing lines between sections. The pipe decorators (|) are added automatically by CSS.

Note: The footer shows the quote source. Each section is separated by a horizontal line with decorative pipe connectors.

18. FLOW DIAGRAM

Create process flow diagrams with boxes and arrows. Use alien-flow-row for horizontal flows and stack rows vertically for multi-step processes.

<div class="alien-flow-diagram"> <!-- Row 1 --> <div class="alien-flow-row"> <div class="alien-flow-node">Grab dice</div> <div class="alien-flow-arrow-right">→</div> <div class="alien-flow-node">Modify</div> </div> <!-- Row 2: Corner arrow left and node --> <div class="alien-flow-row"> <div class="alien-flow-arrow-corner-left">↲</div> <div class="alien-flow-node">Add stress</div> </div> <div class="alien-flow-arrow-down">↓</div> <!-- Row 3 --> <div class="alien-flow-row"> <div class="alien-flow-node">Roll dice</div> </div> <div class="alien-flow-arrow-down">↓</div> <!-- Row 4: Three outcomes with lines --> <div class="alien-flow-row"> <div class="alien-flow-node">Failure</div> <div class="alien-flow-line">—</div> <div class="alien-flow-node">Success</div> <div class="alien-flow-line">—</div> <div class="alien-flow-node">Panic</div> </div> </div>
Note: Build flow diagrams by stacking rows vertically. Available connectors:
  • alien-flow-arrow-right with → for horizontal arrows (left to right)
  • alien-flow-arrow-down with ↓ for vertical arrows (top to bottom)
  • alien-flow-arrow-corner-left with ↲ for turning down and to the left
  • alien-flow-arrow-corner-right with ↳ for turning down and to the right
  • alien-flow-line with — for connecting branching outcomes
Use corner arrows to show flow turning from one node to another. Use lines (—) to connect multiple outcome boxes showing they are all results of the branching.

18b. FLOW DIAGRAM (MERMAID ALTERNATIVE)

Mermaid is a JavaScript library that renders diagrams from simple text definitions. More concise syntax with automatic layout.

<!-- Include Mermaid in your HTML head --> <script src="https://cdn.jsdelivr.net/npm/mermaid@10/dist/mermaid.min.js"></script> <script>mermaid.initialize({ startOnLoad: true, theme: 'dark' });</script> <!-- Then use pre tags with class="mermaid" --> <pre class="mermaid"> flowchart TD A[Grab dice] --> B[Modify] B --> C[Add stress] C --> D[Roll dice] D --> E[Failure] D --> F[Success] D --> G[Panic] </pre>
Mermaid Syntax Tips:
  • flowchart TD = top-down flow (use LR for left-right)
  • A[Text] = rectangular node
  • A{Text} = diamond (decision) node
  • A([Text]) = stadium-shaped node
  • --> = arrow, --- = line without arrow
  • -->|label| = arrow with label
⚠ Foundry VTT Warning: Mermaid requires JavaScript to render. Foundry VTT journal entries may restrict JS execution for security. Test compatibility before using in production. The CSS-based flow diagram (#17) is safer for Foundry use.

WORKFLOW TIPS

Creating Your Chapter Files:

  1. Copy chapter-template.html for each chapter
  2. Name them like: 01_Chapter_Title.html, 02_Next_Chapter.html, etc.
  3. Open your PDF and this cheat sheet side-by-side
  4. Copy text from PDF and paste into your HTML file
  5. Wrap content with appropriate HTML elements from this cheat sheet
  6. Update the metadata comments at the bottom (Chapter Number, Source Pages, Sort Order)
  7. Save and preview in your web browser

Importing into FoundryScribe:

  1. Place all your completed HTML files in a directory
  2. Include the alien-rpg.css file in the same directory
  3. Run FoundryScribe and select "Import from HTML"
  4. Point it to your directory
  5. Export to Foundry VTT JSON format

Best Practices:

  • Keep your HTML files numbered so they import in the correct order
  • Use semantic HTML - don't just wrap everything in divs
  • Preview frequently in a web browser to check formatting
  • Save your work often!
  • Keep the original PDF handy for reference
